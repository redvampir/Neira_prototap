# Инструкции для Codex CLI (локальные правила проекта)

## Язык и тон
- Пиши объяснения, комментарии и документацию на русском.
- Английские термины допускаются, если они стандартны де-факто, либо с кратким пояснением.
- Не задавай вопросов в стиле «Хотите ли вы, чтобы я продолжил?» — если задача ясна, делай.

## Процесс
- Работай циклом PLAN → IMPLEMENT → TEST → REFACTOR (планируй внутри; наружу показывай кратко по необходимости).
- Запрещено оставлять `TODO`, заглушки, `pass` и «допиши сам».

## Код
- «Less is More»: минимальная сложность, максимум читаемости.
- Ориентир: функции ~20–60 строк, вложенность не глубже 3 уровней.
- Для Python используй type hints, валидацию входных данных и обработку ошибок.
- Думай как атакующий: ввод недоверенный, учитывай права доступа и возможные утечки.

## Проверка
- Любые правки подтверждай тестом или скриптом проверки, плюс пример использования и разбор edge cases.

## Документация
- Если добавился новый модуль/фича — обнови `README.md` (инструкции оформляй чек‑листами).

## UE4 C++ (если когда-нибудь понадобится)
- Для `UObject` не используй сырые указатели: `TObjectPtr`, `TWeakObjectPtr`, `TSharedPtr`.
- Категории/DisplayName — на русском; строки через `TEXT("...")`; логи через `DEFINE_LOG_CATEGORY_STATIC`.
